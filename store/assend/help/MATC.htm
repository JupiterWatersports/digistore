MATC Manual

Introduction

This is the contribution MATC (customer must MUST AGREE to TERMS and CONDITIONS). With this contribution installed, the customer MUST AGREE to TERMS and CONDITIONS when creating an account and/or when buying products. WHEN the customer must agree can be decided from a new menu in the admin area of the store. admin Setup > Accept Terms & Conditons menu

The customer must check a checkbox to continue. If the checkbox is not checked the continue button will have opacity and be disabled. If a customer hoovers the disabled button the checkbox will light up in a red color to indicate what needs to be done. Some demo screenshots is included...

Examples

Using MATC

When MATC is first installed your site will not look or work different than a base osCommerce site. You will need to configure MATC to take advantage of its features. To activate the MATC features you need to go to the admin panel and make changes in the new configuration menu "Terms & Conditions".

For a standard OSCommerce store:
Set "Must accept when registering" = true and/or "Must accept at checkout" = true.
Set "Textarea - Show?" = true if you would like to display the textarea.
This will make it work out of the box as the other configuration options is adaped for a standard osCommerce installation.

"Textarea - Mode (How to get the contents)" should always be set to "Returning code" if you do not have any other contribution installed. Set it to "SQL" if you would like to load the textarea contents from the database. The SQL code provided will load a product's description and this is of cource just an example.

Using it with digistore inbuilt CMS

Set "Textarea - Mode" to SQL.
Set "Textarea - SQL" to "SELECT si_content AS thetext FROM information WHERE language_id = ".$languages_id." AND si_id = X;"

Set the X in the SQL code to the approperiate value for your site. in this case it will be 2 unless you deleted condtions then re-added to find the id goto information box and hover over conditions, you will see a url like this

http://localhost/catalog/conditions-of-use-i-2.html 2 is the id value
